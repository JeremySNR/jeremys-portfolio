## What it is

Foundry is a governance layer that sits between your tickets and the AI agents that do the coding. It takes a raw work request all the way through to a reviewed pull request, and keeps a human in control the whole way. The idea is simple: let agents do the heavy lifting, but make every decision along the way explicit and safe.

## Why I built it

Frontier models can write code on their own for hours now, so raw capability isn't really the bottleneck anymore. Everything around the agent is. Was the ticket actually ready? Which repo does this belong in? Is this change safe to hand off? Who signed off, what did it cost, and why did the agent do that? Foundry answers all of those by design instead of after something has gone wrong.

## How it works

A ticket comes in from your tracker and Foundry runs it through a clear pipeline:

- Reads the ticket and checks it's actually ready to work on
- Pulls in the context that matters (the right repo, files, and tests)
- Classifies the risk, flagging anything that touches auth, payments, or personal data
- Plans the work and pauses for a human to approve it
- Dispatches the agent, and retries sensibly if CI fails
- Hard-blocks the scary stuff. Forbidden paths, migrations, and auth changes never run on their own

## The good bits

- Works with what you already use: Linear, GitHub Issues, or Jira for tracking, GitHub or GitLab for code, and Cursor, Claude Code, or webhooks for the agents
- A default-deny policy engine written in both Python and OPA/Rego, with tests required before any policy change ships
- A full audit trail with content hashing, and evidence packs that map to SOC 2, ISO/IEC 27001, and the EU AI Act
- A dashboard for the whole fleet: delivery trends, epic views, and scorecards showing how each agent does by provider and repo
- A human has to approve before anything runs, with role-based access, replay protection, and budget caps per run

## Under the hood

Python and FastAPI, SQLAlchemy (SQLite for dev, Postgres in production) with Alembic migrations, an OPA/Rego policy engine, durable orchestration (Temporal is on the way), OpenTelemetry for observability, and a Docker Compose setup. It's Apache-2.0 licensed.
