## Overview

Project Foundry is the governance layer for autonomous AI coding agents. It sits between raw work requests and the agents that implement them, automating the journey from ticket to reviewed pull request while keeping humans firmly in control. Built for the era of long-horizon agentic engineering, where models can work autonomously for extended periods, Foundry makes every previously invisible decision explicit and governed.

## The Problem

Frontier models can now code autonomously for hours, but raw capability is no longer the bottleneck. The gap is everything surrounding the agent: was the ticket actually ready? Which repository does this belong in? Is the change safe to delegate? Who signed off? What did it cost? And why did the agent do what it did? Foundry answers these questions by design rather than after the fact.

## How It Works

A tracker webhook hits the Foundry API, which drives an orchestrator through a clear pipeline:

- **Analysis** — readiness assessment, missing acceptance criteria
- **Context gathering** — relevant repo, files, and tests
- **Risk classification** — flagging auth, payments, or PII exposure
- **Delivery planning** — with a mandatory human approval gate
- **Governed agent dispatch** — with retry loops on CI failures
- **Hard policy blocks** — forbidden paths, migrations, and auth changes never auto-run

## Key Features

- **Multi-vendor by design** — trackers (Linear, GitHub Issues, Jira), SCMs (GitHub, GitLab), and agents (Cursor, Claude Code, webhooks, manual)
- **Default-deny policy engine** — written in both Python and OPA/Rego, with tests required for every policy change
- **Complete audit trail** — content-hashed evidence packs mapped to SOC 2, ISO/IEC 27001, and EU AI Act controls
- **Fleet visibility** — a dashboard with delivery trends, epic views, and agent scorecards showing success rates by provider and repo
- **Safety guarantees** — mandatory human approval before any dispatch, RBAC for sensitive areas, webhook dedup/replay protection, and per-run budget caps

## Technology

Python 3.11+, FastAPI, SQLAlchemy (SQLite in dev, Postgres in production) with Alembic migrations, an OPA/Rego policy engine, durable orchestration (with Temporal in progress), OpenTelemetry observability, and Docker Compose deployment. Licensed Apache-2.0.
